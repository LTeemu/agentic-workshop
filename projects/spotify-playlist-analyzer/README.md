# Spotify Playlist Analyzer

Analyze your Spotify playlist export CSV ‚Äî **Python** for data analysis, **Flask** for the dashboard.

Record label aesthetic with warm blacks, browns, and gold.

## Quick Start

### Prerequisites

- Python 3.10+ with `pip`

### 1. Drop your CSV

Place your Spotify playlist CSV in `data/spotify-playlist.csv`.  
A sample dataset is at `data/sample.csv` for testing.

The CSV can be tab-separated or comma-separated (auto-detected). Column names are matched case-insensitively.

| Column                                               | Description                     |
| ---------------------------------------------------- | ------------------------------- |
| Song                                                 | Track name                      |
| Artist                                               | Artist name(s), comma-separated |
| BPM                                                  | Beats per minute                |
| Camelot                                              | Camelot wheel notation          |
| Energy                                               | 0‚Äì100                           |
| Added at                                             | Date added to playlist          |
| Duration                                             | mm:ss                           |
| Popularity                                           | 0‚Äì100                           |
| Genres                                               | Comma-separated tags            |
| Album                                                | Album name                      |
| Album Date                                           | Release date                    |
| Dance, Acoustic, Instrumental, Valence, Speech, Live | 0‚Äì100                           |
| Loud (db)                                            | Decibels                        |
| Key                                                  | Musical key                     |
| Time Signature                                       | Time signature                  |
| Spotify Track Id                                     | URI                             |
| ISRC                                                 | Recording code                  |
| Explicit                                             | yes / no                        |

### 2. Install & run

```bash
cd python
pip install -r requirements.txt
python server.py
```

The server will:

1. Check if `analysis.json` is missing or older than your CSV
2. Auto-run `analyze.py` if needed
3. Start the dashboard at **http://localhost:5000**

Or from the project root:

```bash
npm start
```

### Manual commands

```bash
cd python
python analyze.py ../data/spotify-playlist.csv -o ../data/analysis.json
```

**Dev iteration** ‚Äî process only first N rows for faster testing:

```bash
python analyze.py ../data/spotify-playlist.csv -o ../data/analysis.json --limit 10
```

### Iteration

- **Edit code** ‚Üí Flask reloads automatically in debug mode
- **Update CSV** ‚Üí restart the server (it auto-rebuilds on boot)
- **Re-run analysis** ‚Üí page refresh picks up new data (mtime cache)
- All `avg_*` stats use **median** (not mean) for robustness
- Popularity=0 is treated as missing data (696 of 3816 tracks)

## How It Works

```
spotify-playlist.csv
        ‚îÇ
        ‚ñº  (python/analyze.py)
  data/analysis.json          ‚Üê full structured analysis
        ‚îÇ
        ‚ñº  (python/server.py reads JSON, caches by mtime)
  Flask @ localhost:5000
        ‚îÇ
        ‚ñº  (Chart.js in browser)
  Interactive dashboard
```

The Python analyzer computes seven analysis categories and writes a single JSON file. The Flask server reads that file (with mtime-based caching ‚Äî re-reads only when the file changes) and renders Chart.js visualizations with per-bar HSL colors.

## Analysis Categories

| Category           | What it includes                                                                                                                                                                                                        |
| ------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Basic Stats**    | Total songs, unique artists/genres/albums, median BPM/energy/dance/valence/duration/popularity, explicit count                                                                                                          |
| **Audio Features** | Bucketed histograms for BPM, Energy, Dance, Valence, Acousticness, Instrumentalness, Speechiness, Liveness, Loudness                                                                                                    |
| **Genres**         | Top 20 genres with counts and percentages, total unique genres                                                                                                                                                          |
| **Artists**        | Top 20 artists, collaboration count, per-artist genre tags                                                                                                                                                              |
| **Mood & Key**     | Key distribution, Camelot wheel, mood quadrants (valence √ó energy), interactive scatter plot                                                                                                                            |
| **Timeline**       | Songs added by year/month, album release year distribution                                                                                                                                                              |
| **Perfect Track**  | Mode (most common) value for each audio feature, composite attributes (key, camelot, genre, time signature), closest real match (most representative song) and furthest real match (biggest outlier) from your playlist |
